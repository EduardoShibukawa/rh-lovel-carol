# AUTHORITIES - Referência Central de Autoridades de Recrutamento

## Overview

Este arquivo serve como referência central para todas as autoridades de recrutamento utilizadas no projeto rh-carol. Cada skill individual faz referência a esta autoridade conforme apropriado.

---

## 1. Lou Adler (Performance-based Hiring)

**Especialidade:** Performance-based Hiring (PBH)  
**Referência:** https://performancebasedhiring.com/

### Princípios Oficiais
- **Impacto 90 dias**: Definir trabalho como resultados (impacto), não lista de skills
- **Career growth**: Top talentos não mudam por cargos laterais, mas por 30% de aumento no desafio/impacto
- **Outcome vs Skills**: Resultado > competências técnicas

### Aplicação no Projeto
- **POST**: Hooks focados em impacto de 90 dias
- **OUTREACH**: Mencionar impacto que o candidato terá nos primeiros 3 meses

---

## 2. Stacy Zapar (3x3 Rule)

**Especialidade:** High-Response Outreach  
**Referência:** LinkedIn Learning - "Optimizing Your Candidate Outreach Strategy"

### Princípios Oficiais
- **3x3 Rule**: 3 mensagens, 3 dias entre cada (máx 300 caracteres no LinkedIn)
- **Personalização**: 3 pontos reais do perfil do candidato
- **Candidate as Client**: Se a mensagem for genérica, a resposta será nula

### Aplicação no Projeto
- **OUTREACH M1**: Máximo 300 caracteres
- **OUTREACH Follow-up**: Day 4 (nudge), Day 7 (breakup)
- **Personalização**: 3 exemplos do que buscar no perfil

---

## 3. Glen Cathey (Boolean Black Belt)

**Especialidade:** Sourcing Science  
**Referência:** https://www.booleanblackbelt.com/

### Princípios Oficiais
- **5+ Sinônimos**: Expandir cada termo para 5+ sinônimos
- **X-Ray Search**: site:linkedin.com/in para candidatos invisíveis
- **Text-poor profiles**: Candidatos com perfis simples também são talento
- **Dark matter**: 80% dos talentos estão ocultos sob cargos não-padrão
- **Iterative search**: Run search → Analyze results → Refine

### Aplicação no Projeto
- **HUNTING**: Boolean strings com 5+ sinônimos, X-Ray, exclusões

---

## 4. Aaron Ross (Predictable Revenue)

**Especialidade:** Outbound Sales & ICP  
**Referência:** https://predictablerevenue.com/

### Princípios Oficiais
- **ICP com hipóteses**: Estruturar ICP como hipóteses testáveis (Targeting, Need, Solution)
- **Seed/Net/Spear**: Tipos de prospect (Seed=referral, Net=inbound, Spear=cold outbound)
- **SDR/BDR specialization**: Diferença entre SDR (qualificação) e BDR (descoberta)
- **Cold Calling 2.0**: Abordagem moderna de outbound

### Aplicação no Projeto
- **HUNTING**: ICP estruturado para vagas comerciais

---

## 5. Gergely Orosz (Pragmatic Engineer)

**Especialidade:** Tech Hiring & Developer Mindset  
**Referência:** https://blog.pragmaticengineer.com/

### Princípios Oficiais
- **Senioridade = Autonomia/impacto**: Não tempo de casa
- **Role definition**: JD deve definir sinais, não apenas competências
- **Tech stack específica**: Desafios reais da stack, falar a língua do desenvolvedor
- **Proibir buzzwords**: "inovador", "dinâmico", "visionário" são vazias

### Aplicação no Projeto
- **HUNTING**: Foco em impacto técnico, não tempo de experiência
- **POST**: Desafios técnicos específicos
